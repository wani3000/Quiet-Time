# iOS 알림 실기기 테스트 체크리스트

## 1) 사전 준비
- 테스트 기기: iOS 16+ 권장
- 앱 설치 방식: Debug 또는 TestFlight
- 기기 설정에서 앱 알림 권한 초기화(설정 > 알림)

## 2) 기본 권한/토글
- 앱 최초 진입 후 알림 권한 팝업 노출 확인
- `알림설정 > 알림 받기` ON 시 권한 허용 요청/재요청 흐름 확인
- `알림 받기` OFF 시 예약 알림이 모두 취소되는지 확인

## 3) 시간 설정
- `알림 시간` 모달에서 시간 선택 후 `확인` 시 저장되는지 확인
- 모달을 아래로 내려 닫았을 때(확인 미클릭) 변경이 저장되지 않는지 확인
- 24h/AM-PM 표기 전환 시 UI 표기만 바뀌고 실제 예약 시각은 동일한지 확인

## 4) 주기 설정
- 매일: 다음 알림이 다음 도래 시각으로 예약되는지 확인
- 매주: 선택한 요일/시각 기준으로 반복되는지 확인
- 매달: 선택한 날짜/시각 기준으로 반복되는지 확인
- 매달 29~31일 설정 시 말일 보정 동작(30일/2월) 확인

## 5) 백그라운드/종료 상태
- 앱 백그라운드 상태에서 알림 수신 확인
- 앱 완전 종료 상태에서 알림 수신 확인
- 알림 탭 시 홈 화면(`/`)으로 이동되는지 확인

## 6) 변경 반영 회귀
- 시간만 변경 후 기존 예약이 갱신되는지 확인
- 주기만 변경 후 기존 예약이 갱신되는지 확인
- ON/OFF를 빠르게 반복해도 중복 예약이 쌓이지 않는지 확인

## 7) 권장 확인 로그
- Xcode Console에서 `알림 스케줄링 완료` 로그 확인
- `NotificationService.checkScheduledNotifications()` 호출 시 예약 1건만 존재하는지 확인
